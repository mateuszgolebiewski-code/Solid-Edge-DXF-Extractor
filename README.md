# SolidEdge DXF Extractor
Author: **Mateusz Gołębiewski** ([www.mateuszgolebiewski.pl](http://www.mateuszgolebiewski.pl))

An application (operating entirely within the Windows .NET environment) that flawlessly automates the process of extracting clean `.dxf` files used in CNC cutting environments (PLASMA CUTTERS, LASERS), exporting flat patterns directly from Solid Edge assemblies.

## Requirements

- **Solid Edge** ST10 or newer (installed on the computer)
- **.NET 8.0 Runtime** (or newer) for desktop
- Operating system: Windows 10/11 (64-bit)


## How to use (Workflow Instruction)

1. Launch the **SolidEdge DXF Extractor** program (`SolidEdge_FlatExporter.exe`). You don't need to manually start Solid Edge at this point; our program will automatically establish the appropriate machine connection via COM.
2. Click **"Browse..."** and precisely specify the main assembly file (`.asm`). **Alternatively:** you can simply drag and drop the `.asm` file from the file explorer onto the program window, or directly onto the `SolidEdge_FlatExporter.exe` shortcut - this bypasses the need to use the button and automatically starts the assembly scanning!
3. Press **"Scan"** (if you didn't use the drag and drop option). The program will efficiently and thoroughly analyze the list of subassemblies in the background, counting all sheet metal parts generated by the "Sheet Metal" interface. It will recognize their material and extract thicknesses from variables.
4. Select the parts from the generated interactive list.
5. Choose whether you want to keep or ignore (remove) bend lines for the generated files.
6. Specify your custom file naming scheme - whether you wish to include a thickness prefix and a specific material suffix.
7. Finally, choose **"Export"**!

> **IMPORTANT SAFEGUARD METHODS**: The program features an analytical engine that forces the creation of missing FlatPatterns on the fly and removes junk header blocks generated by the SolidEdge software, protecting the system from outputting zero-vector empty `.dxf` files (the so-called "32-kilobyte DXF bug"). Additionally, the application categorizes selected layers and programmatically removes `[UP/DOWN]_CENTERLINES` vectors straight from the text file syntax, leaving a flawless smooth contour for DXF reading programs. Assembly files are naturally neither overwritten nor visually modified.

## Generated Files Format (Naming Scheme)

```
[Thickness_][Material_]BasePartName.dxf
```
Example: `2.0mm_DC01_SolidBracket_v1.dxf`

## Automated Troubleshooting in Engine Logic
| Event | Engine Procedure / Message |
|---------|-------------|
| "Solid Edge is not installed" | Ensure that the `Solid Edge` engine with API is correctly registered in the Windows COM Registry. |
| Thickness = "unknown" | The analyzer did not find an equivalent environment variable (`Material Thickness` or `Gage`). |
| 34KB DXF Case | The SE API threw an empty skeleton. The engine will delete the fake volume, steer the working parameters, and re-invoke the "Flatten and Dump" operation. |
| Remnant layer artifacts | The DXF block format parser intervenes by cutting out objects with headers that SE uses for `Centerlines` from the entity list. |
